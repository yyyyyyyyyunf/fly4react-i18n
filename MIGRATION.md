# Migration from `react-native-localize-ext`

`@fly4react/i18n` is a ground-up redesign of `react-native-localize-ext`. The API surface, package names, and architecture have all changed.

## Package names

| Old                                | New                     |
| ---------------------------------- | ----------------------- |
| `react-native-localize-ext`        | `@fly4react/i18n-core`  |
| `@react-native-localize-ext/core`  | `@fly4react/i18n-core`  |
| `@react-native-localize-ext/react` | `@fly4react/i18n-react` |

## API changes

### Before: Proxy chain API

```ts
import { createI18n } from '@react-native-localize-ext/core';

const i18n = createI18n({
  defaultLocale: { key: 'en', values: en },
});

i18n.greeting.hello({ name: 'World' });
```

### After: function-based API

```ts
import { createI18n } from '@fly4react/i18n-core';
import { useTranslation } from '@fly4react/i18n-react';

const i18n = createI18n({
  defaultLocale: 'en',
  messages: { common: en },
});

function Component() {
  const { t } = useTranslation('common');
  return <div>{t('greeting.hello', { name: 'World' })}</div>;
}
```

## Key differences

1. **No more Proxy chain.** Use `t('key', values)` instead of `i18n.a.b.c(values)`.
2. **ICU MessageFormat.** The old `{{name}}` syntax is replaced by `{name}` and full ICU support.
3. **JSON source of truth.** Messages are stored in JSON files; TypeScript types are generated by the CLI.
4. **Namespaces.** Messages are organized by namespace (one JSON file per namespace).
5. **React 19 / Suspense.** The React integration uses hooks and Suspense by default.
6. **SSR / RSC.** A dedicated `@fly4react/i18n-server` package supports server components.

## Migration steps

1. Move your translation objects from `.ts` files to `.json` files under `locales/<locale>/`.
2. Run `npx fly4react-i18n generate` to generate types.
3. Replace `createI18n({ defaultLocale: { key, values } })` with `createI18n({ defaultLocale, messages })`.
4. Replace `I18nProvider(...instances)(Component)` HOC with the new `<I18nProvider i18n={i18n}>` component.
5. Replace Proxy chain access with `t('namespace.key', values)`.
6. Update interpolation syntax from `{{name}}` to `{name}`.
